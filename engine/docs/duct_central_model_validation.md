# DUCT_CENTRALモデル検証メモ（pyhees整合）

このドキュメントは `acmodel::DuctCentralModel` の実装を、pyhees（主に 4_2_a / 4_2_b / 4_2）に合わせるための実装方針と、回帰テストの観点をまとめたものです。

## 1. 検証対象と単位

- 実装: `engine/acmodel/duct_central_model.cpp`
- 回帰テスト: `engine/solver/tests_cpp/test_acmodel_core.cpp`
- 主要単位:
  - 能力 `q_*`: `[W]`
  - 電力 `E_E_*`: `[kW]`（= `kWh/h` と数値的に同値）
  - 風量 `V_*`: モデル内部は `[m3/h]`（入力 `InputData` は `[m3/s]`）

## 2. pyhees整合の主要ポイント

### 2.1 送風機電力（式(37)(38)）

`calculateFanPower()` は以下の形を採用:

- `E_fan = (P_fan_rtd - f_SFP * V_vent) * ((V_supply - V_vent)/(V_dsgn - V_vent)) * 1e-3`
- `f_SFP = 0.4 * 0.36`
- `q_hs > 0` のときのみ加算、最終的に `max(0, ...)`

### 2.2 デフロスト境界（式(24)）

暖房のデフロスト補正は pyhees 条件に合わせて:

- `Theta_ex < 5`
- `h_ex > 80`

のとき `C_df_H = 0.77`、それ以外は `1.0`。

### 2.3 冷房の顕熱伝達率（式(36a)(36b)）

冷房側の `alpha_c_hex_C` は pyhees と同じ式（不要な係数拡大なし）:

- `alpha_dash_c_hex_C = 0.050 * log((max(V,400)/3600)/A_f_hex) + 0.073`
- `alpha_c_hex_C = alpha_dash_c_hex_C * (c_p_air + c_p_w * X_in)`

この部分は過去にスケーリング差（`*1000`）がずれ要因になったため、回帰で監視する。

## 3. V_ventの扱い

- `InputData.V_vent` は `V_outer` と独立入力（換気分）
- モデル既定値は `0 m3/h`
- `V_vent = 0` は「有効入力」として扱う（未入力扱いにしない）

## 4. テスト戦略（現行）

`test_acmodel_core.cpp` の DUCT_CENTRAL ブロックで以下を確認:

- 暖房/冷房の代表点で合計電力回帰
- 送風機電力と圧縮機電力をログから分離して個別回帰
- `V_outer` を変えても結果不変（`V_vent` と分離されていること）
- `V_vent=0` 明示と未指定相当の同値
- デフロスト境界（`h_ex=80%` と `>80%`）

### テスト専用ルール（本体非適用）

ユーザー検証条件の再現のため、テスト内に限って:

- `V_supply < 160 m3/h` の場合に `V_vent = 160 m3/h` を与える

というケースを追加している。これは本体実装の仕様ではなく、比較検証のためのテスト条件。

## 6. 変更時チェックリスト

- DUCT_CENTRALを変更したら必ず `vtsimnx_solver_cpp_test_acmodel_core` を実行
- 送風機・圧縮機の内訳回帰が両方通ることを確認
- 風量単位（`m3/s` 入力、`m3/h` 内部）を再確認
