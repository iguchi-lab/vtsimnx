# 外壁表面が外気温より低くなる場合のチェックリスト

日射・夜間放射なしで冬期暖房時に「外壁側表面 < 外気温」となる場合の確認項目です。

---

## 1. 日射・夜間放射が本当に無効か

### builder のフラグ

- raw 設定で **`add_surface_solar: false`** と **`add_surface_nocturnal: false`** を指定しているか確認。
- 未指定時は **true** になるため、外壁に日射/夜間放射ブランチが追加される可能性がある。

### 表面データのキー

- 日射を無効にする: その表面に **`solar` キーを付けない**（`add_surface_solar: false` なら日射ループ自体がスキップされる）。
- 夜間放射を無効にする: その表面に **`nocturnal` / `night_radiation` キーを付けない**（`add_surface_nocturnal: false` なら夜間放射ループ自体がスキップされる）。
- **`nocturnal` に 0 以外が入っていないか**（テンプレートの残りなど）も確認。

---

## 2. ソルバ入力（built config）の確認

シミュレーションに渡している **thermal_branches** で、該当外壁の室外側表面ノード（例: `外部-LD(S_外壁)_wall_s`）を確認する。

- **void → 室外側表面** のブランチ  
  - `subtype: "solar_gain"` または `"nocturnal_loss"` の **void→o_s** が 1 本も無いこと。  
  - 日射・夜間放射を無効にしているなら、ここに heat_generation が乗ったブランチは存在しない想定。
- **室外側表面 → 外気（end_node）** のブランチ  
  - `subtype: "convection"` で **o_s → end_node** が 1 本あること。  
  - `conductance` は 面積×α_o（正の値）であること。

---

## 3. 熱回路の向き（builder 側）

- 室外側対流は **「室外側表面 → 外気」** の 1 本（`node_names[-1] -> end_node`）。
- 熱流は Q = G×(T_表面 − T_外気)。表面の方が高温なら外気へ熱が流れる向きで正しい。

---

## 4. 読んでいるノードの確認

- 見ている温度が **室外側表面ノード**（o_prefix_s、例: `外部-LD..._wall_s`）の値か確認。
- 層の境界ノード（例: `5-6`）や室内側表面と取り違えていないか。

---

## 5. 過渡の可能性

- 外気温が上昇している時間帯では、壁の熱容量のため「外気温 > 室外側表面」になることはあり得る（外気が表面を暖めている過渡状態）。
- 定常に近い「暖房のみ」の状態なら、通常は 室温 > … > 室外側表面 > 外気温 の順になる。

---

## 6. 推奨設定（日射・夜間放射を使わない場合）

raw 設定のトップレベルまたは builder オプションで次を明示することを推奨します。

```json
{
  "add_surface_solar": false,
  "add_surface_nocturnal": false
}
```

これで日射・夜間放射のループが走らず、該当する void→表面 ブランチは追加されません。
