### 計算モデルの物理・数学メモ（VTSimNX）

このドキュメントは、VTSimNX の **熱・換気の計算が何を解いているか**、および改修時に壊れやすい **符号/単位/離散化/安定性**のポイントをまとめたものです。

---

### 1. 基本：ネットワークの熱収支

熱ネットワークはノード温度 \(T\) を未知数とし、各ノードで「流入熱量の和 = 0」を満たすように組み立てます（内部は \([W]\) ベース）。

- **伝熱（conductance）**: \(q = G\,(T_s - T_t)\)（\(G\): \([W/K]\)）
- **発熱（heat_generation）**: \(q\) を外力として加える（\([W]\)、スカラー/時系列）
- **移流（advection）**: 換気流量により \(q \approx \rho c_p |\dot V| (T_{up} - T_{down})\)

どの項が「右辺（既知）」に入り、どの項が「係数行列（未知温度）」に入るかは **calc_t の有無**で変わります。

---

### 2. 単位の約束（特に混ざりやすい点）

- solver内部の熱収支は **\[W]**
- RC法（層分割）:
  - 伝熱ブランチ `conductance`: **\[W/K]**（builderが面積込みで生成）
  - 層ノードの `thermal_mass`: **\[J/K]**
- 応答係数法（CTF / `response_conduction`）:
  - 係数は **面積あたり（per m²）**で扱う（option B）
  - `resp_a_*`, `resp_b_*`: \([W/m^2/K]\)
  - `resp_c_*`: 無次元
  - solver側で温度項に **面積 \(A\)** を掛けて \([W]\) に変換する（よって `area` 必須）

換気（風量）の単位について（重要）:

- 換気回路網の solver が扱う風量は **体積流量 \(\dot V\) \([m^3/s]\)** です
  - `ventilation_branches[].vol`（`fixed_flow`）も \([m^3/s]\) として扱います
- 熱の移流では \(\rho c_p \dot V\) を使って \([W]\) に変換します
- 湿度（絶対湿度 \(x\)）の移流では、\(\rho \dot V\) により **質量流量 \([kg/s]\)** に変換して扱います（\(\rho\) は乾燥空気密度の近似）
- 濃度（\(c\)）の移流は、old_vtsim 互換として **体積流量 \([m^3/s]\)** ベースで扱います（\(\dot V/V\) が \([1/s]\) になるため）

---

### 3. RC法（層分割）の数学的ポイント

壁を \(n\) 層のRC連鎖として離散化します。

- 伝導: 各層は \(G_i = A\,\lambda_i/t_i\)
- 熱容量: 各層の容量は \(C_i = A\,v\_capa_i\,t_i\)
- 層ノードの `thermal_mass` は等価的に表面/内部へ配分されます（builder実装に従う）
- 数値的には「分割数」「timestep」によって高周波成分が減衰しやすい（数値拡散）

---

### 4. 応答係数法（CTF / response_conduction）の数学的ポイント

壁内部の状態をネットワークに展開せず、両端表面の熱流を ARX 形式で表現します。

#### 4.1 形式（熱流密度 \(q''\)）

src側（source表面）:

\[
q''_{src}(k) =
\sum_{j=0}^{m} a^{src}_{j}\,T_{src}(k-j)
\;+\;
\sum_{j=0}^{m} b^{src}_{j}\,T_{tgt}(k-j)
\;+\;
\sum_{i=0}^{m-1} c^{src}_{i}\,q''_{src}(k-1-i)
\]

tgt側も同様に \(T_{tgt}\) と \(T_{src}\) を入れ替えた形を持ちます。

solverでは \(q = A\,q''\) に変換して熱収支（\[W]）に入れます。

#### 4.2 初期条件

履歴（過去温度・過去熱流）をどう埋めるかで初期過渡が変わります。
現実装では初期温度から履歴温度を埋め、熱流履歴は 0 から開始します。

#### 4.3 安定性（sum(c)≈1）

非常に遅い系では \(\sum c \to 1\) に近づき、丸め誤差で不安定化しやすい。
builderはこのケースで **定常U値（メモリなし）へフォールバック**することがあります。

---

### 5. ソルバ（線形系）のポイント

熱計算は基本的に線形方程式 \(A\,T=b\) を解きます。

- **直接法（Sparse）**での解法を採用
- 性能のため、係数行列のパターン/分解をキャッシュし、必要時に再構築します
- 改修時に壊れやすいのは以下:
  - **符号**（枝の向き、熱流の正方向、RHSへ移項する符号）
  - **単位**（per m² / 面積込み、\[W]と\[W/m²]の混在）
  - **既知/未知温度の扱い**（calc_t=false の既知温度項は RHS に吸収される）

### 7. 湿度（絶対湿度 x）・濃度（c）の時間発展（メモ）

#### 7.1 湿度 \(x\)（無次元: kg/kg(DA)）

換気移流＋生成（発湿）を考えると、概念的には

- 生成: \(\dot m_x\,[kg/s]\)
- 移流: \(\rho \dot V\,[kg/s]\)

で、各室の更新は 1 タイムステップで陰解法（Gauss-Seidel）により安定に解きます。

補足（\(\rho\) の扱い）:

- 現状の湿度移流では \(\rho\) を **乾燥空気密度の定数**として扱っています（温度・湿度依存の密度変化は未反映）。
- 将来的に精度が必要になった場合、\(\rho(T)\) 等へ拡張する余地があります。

#### 7.2 濃度 \(c\)（例: count/m³）

old_vtsim 互換の 1 室モデルは

\[
\frac{dc}{dt}=k_1-k_2 c
\]

として、厳密解

\[
c(t+\Delta t)=(c(t)-k)\exp(-k_2\Delta t)+k,\quad k=\frac{k_1}{k_2}
\]

を用います。ここで

- \(k_2=\beta+\sum \dot V_{out}/V\)
- \(k_1=m/V+\sum (1-\eta)\,c_{src}\,\dot V_{in}/V\)

（\(\beta\): 沈着係数 \([1/s]\), \(m\): 発塵量 \([count/s]\), \(\eta\): 除去効率）

### 8. 参考：CRIEPI冷房モデル（エアコン熱源特性）

acmodel の `CRIEPIModel` は、CRIEPI の熱源特性モデルの考え方（冷房運転）を背景として、仕様書データから機器固有パラメータを同定し、任意条件で COP/電力を推定します。

- 基本式: \(COP = Q/P\)
- 消費電力の分解: \(P = P_{comp} + P_{aux}\)
- 入力として温湿度（室内外）・風量などを扱う（記号例: \(T\), \(X\), 風量）

参考: [家庭用エアコンの熱源特性モデルの開発（その1：冷房運転時モデル）](https://doi.org/10.18948/shase.38.190_41)

---

### 9. 参考：RACモデル資料（直吹き壁掛けルームエアコン）

`acmodel` の `RACModel` は、ルームエアコンディショナーの計算手法（章4-3）を背景にしています。
この資料では、外気温 \(\theta_{ex}\) を主要な説明変数とし、吹出風量補正（\(C_{af}\)）、吸込み湿度補正（\(C_{hm}\)）、デフロスト補正（\(C_{df}\)）などの係数で最大出力や消費量を補正する枠組みが整理されています。

単位の注意（資料側）:

- 能力は \(W\)（例: \(q_{rtd}\), \(q_{max}\)）
- 一方で「1時間当たりの処理量/負荷」は \(MJ/h\) を用いる式があり、換算が前提（例: \(\times 3600 \times 10^{-6}\)）

参考: [第四章 暖冷房設備／第三節 ルームエアコンディショナー（Ver.08, 2025.04）](https://www.kenken.go.jp/becc/documents/house/4-3_250401_v08.pdf)
