# 定数・仕様一覧（api builder と core の対応）

ワークスペース統合時に、**api（builder）** と **core（vtsimnx）** で揃えておく定数・単位・仕様を一覧にしたものです。片方だけ変更すると挙動がずれるため、変更時は両方を確認してください。

---

## 1. API（builder）で使用する定数

定義場所: `api/app/builder/surfaces.py`

| 定数名 | 値 | 単位 | 用途 |
|--------|-----|------|------|
| **DEFAULT_ALPHA_I** | 4.4 | W/(m²·K) | 室内側表面の対流熱伝達率 |
| **DEFAULT_ALPHA_O** | 20.3 | W/(m²·K) | 室外側表面の対流熱伝達率 |
| **DEFAULT_ALPHA_R** | 4.7 | W/(m²·K) | 室内表面間の放射熱伝達率（両面の長波放射率 0.9 は既に含む） |
| **DEFAULT_ETA_SW** | 0.8 | — | 短波（日射）の吸収率（外壁日射・ガラス透過日射の床・壁への吸収） |
| **DEFAULT_ETA_LW** | 0.9 | — | 長波の吸収率（夜間放射・発熱の放射配分で表面が吸収するとき） |
| **DEFAULT_EPSILON_LW** | 0.9 | — | 長波放射率（夜間放射の放出側。室内表面間の 4.7 には既に含まれる） |
| **DEFAULT_AIR_V_CAPA** | 1298.0 | J/(m³·K) | 中空層・通気層の空気の体積熱容量（SimHeat で採用の標準値） |

- 室内表面間の放射（4.7）では、両面の放射率 0.9 が既に込まれているため、表面の `eta` は掛けない。
- 日射（短波）には `eta` = 0.8、夜間放射（長波）には `epsilon` = 0.9 を使用する。

## 2. Core（vtsimnx）で定義されている関連定数

### 2.1 建築環境定数（`core/vtsimnx/archenv/archenv.py`）

| 定数名 | 値 | 単位 | 用途 |
|--------|-----|------|------|
| **Air_Cp** | 1.005 | kJ/(kg·K) | 空気の定圧比熱 |
| **Vap_Cp** | 1.846 | kJ/(kg·K) | 水蒸気の定圧比熱 |
| **Vap_L** | 2501.1 | kJ/kg | 水蒸気の蒸発潜熱 |
| **Sigma** | 4.88e-8 | — | ステファン・ボルツマン定数 |
| **Solar_I** | 1365 | W/m² | 太陽定数 |

- **capa_air(v)**: 容積 v [m³] の空気の熱容量 [J/K]。内部で `v * 1.005 * 1.2 * 1000`（= 1206 J/(m³·K)）を使用。  
  → 中空層・通気層の**熱容量**は api では 1298（SimHeat/材料表準拠）を採用しているため、api builder と core の `capa_air` は異なる値になる。日射・夜間放射などの**計算**では core の定数のみ使用。

### 2.2 空気密度（`core/vtsimnx/archenv/archenv.py`）

- **get_rho(θ)**: 空気密度 ρ [kg/m³]。理想気体として `353.25 / (θ + 273.15)`（標準大気圧/気体定数に基づく）。

### 2.3 材料テーブル（`core/vtsimnx/materials/table.py`）

- **lambda**: 熱伝導率 [W/(m·K)]
- **v_capa**: 体積熱容量 [J/(m³·K)]。内部テーブルは [kJ/(m³·K)] で保持し、公開用 `materials` で ×1000 して [J/(m³·K)] に統一。
- **中空層**の v_capa: 内部 1.298 → 公開時 **1298** J/(m³·K)。api の `DEFAULT_AIR_V_CAPA` と一致させる想定。

### 2.4 日射・ガラス（`core/vtsimnx/archenv/solar.py`）

| 引数名 | デフォルト | 用途 |
|--------|------------|------|
| **glass_tau_diffuse** | 0.808 | 拡散日射の透過率（ガラス） |

## 3. 単位の約束（api builder と core で共通）

| 項目 | 単位 | 備考 |
|------|------|------|
| 熱伝導率 lambda | W/(m·K) | 層の `lambda` |
| 厚さ t | m | 層の `t`。mm で渡すと熱容量が約1000倍になるので注意 |
| 体積熱容量 v_capa | J/(m³·K) | 層の `v_capa`、core の公開 `materials` と同じ単位 |
| 熱抵抗 thermal_resistance | m²·K/W | 中空層の `thermal_resistance`（r_value / r） |
| 面積 area | m² | 表面の面積 |
| 放射・日射 | W/m² | core の原則。必要に応じて関数内で換算 |
| 温度 | 摂氏 | core の原則 |

---

## 4. 変更時の確認ポイント

- **空気の体積熱容量**  
  api: `DEFAULT_AIR_V_CAPA`（1298）。  
  core: 材料表の「中空層」v_capa 1.298（公開時 1298）、`capa_air()` は 1.2×1005 で別用途。中空層・通気層の**熱容量**は api の 1298 に合わせる。
- **室内表面間の放射 4.7**  
  両面の長波放射率 0.9 を込めた値。室内表面間の放射コンダクタンスでは表面の `eta` を掛けない。
- **短波 0.8・長波 0.9**  
  日射吸収・夜間放射・発熱の放射配分で api builder が使用。core で日射を計算する場合も同じ考え方で揃えるとよい。
- **材料の v_capa**  
  api の層の `v_capa` と core の公開 `materials` はともに [J/(m³·K)]。core の内部テーブルは [kJ/(m³·K)] なので、公開時 ×1000 を忘れないこと。
